# 🐾 Adams Ark and Avery (3A) – Pet Hotel Management System

## 📌 Project Overview

Adams Ark and Avery (3A) is a digital transformation project for a Dublin-based pet hotel business. The goal of this system is to replace the current paper-based management process with a secure, scalable, and user-friendly web application built using **JavaServer Faces (JSF)**.

The system will support customers, pet attendants, and administrators by streamlining booking, pet care management, and service administration.

## 🎯 Project Objectives

- Digitize customer registration and booking processes
- Store and manage pet care information securely
- Enable role-based access for different user types
- Provide online service browsing and pricing
- Integrate **PayPal** for payment processing
- Ensure system scalability for future expansion (multiple branches)

## 👥 User Roles

### 1️⃣ Customer

Customers can:

- Register and log into the system
- Add and manage pet profiles
- View available services
- Make reservations
- Pay for bookings via PayPal
- View upcoming reservations

### 2️⃣ Pet Attendant

Pet attendants can:

- View pets currently boarding
- Access feeding and play schedules
- View detailed care instructions
- Monitor assigned pets during their stay

### 3️⃣ Admin

Admins can:

- Manage service listings and pricing
- Assign user roles
- Monitor reservations
- Maintain system configuration
- Prepare system for future expansion

## ⚙️ Core Features

### 📝 1. Customer Registration & Authentication

- Secure account creation
- Login and session management
- Role-based access control

### 📅 2. Reservation Management

- Book pet stays with date selection
- Select additional services (e.g., grooming, playtime)
- View and manage bookings

### 🐶 3. Pet Profile Management

- Store pet type, breed, age
- Add special care instructions
- Manage dietary and medical notes

### 💳 4. PayPal Payment Integration

- Secure online payment processing
- Payment confirmation tracking
- Future-ready for additional payment methods

### 📋 5. Care & Schedule Viewing

- Feeding schedule display
- Playtime schedule tracking
- Read-only access for attendants

### 🛡️ 6. System Security

- Role-based access restriction
- Encrypted password storage
- Session timeout handling
- Protection against unauthorized access

### 🌍 7. Scalability

- Designed to support multiple branches
- Modular architecture for adding new services
- Flexible database schema

## 🛠️ Technology Stack

- **Frontend & Backend**: JavaServer Faces (JSF)
- **Language**: Java
- **Server**: Apache Tomcat / Jakarta EE Server
- **Database**: Relational Database (MySQL or similar)
- **Payment Gateway**: PayPal API (Sandbox or Live)
- **Project Management**: JIRA (Agile methodology)
